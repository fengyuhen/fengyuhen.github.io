Object Detection Summrize

Posted on 2018-07-30

目标检测汇总

引入

之前读了目标检测的一些论文，R-CNN，Fast R-CNN，Faster R-CNN，看的时候一知半解，很多地方也马马虎虎，今天对之前的知识做一下梳理

Computer Vision

目前来说，所接触的到CV方向主要有图像分类，目标检测，语义分割，其中目标检测技术的一般流程主要有以下几个方面

  1. region proposal：候选区域的生成
  2. classification：对候选区域提取特征，做物体的分类
  3. bounding box：生成边界框
  4. bounding box regression：边界框回归

Region proposal

Region proposal的生成方式，了解到的有

  1. SS（selective search）：R-CNN，Fast R-CNN中使用
  2. RPN（region proposal network）：Faster R-CNN中使用

RPN

Fast R-CNN利用了GPU，而region proposal方法是在CPU上实现的，这个运行时间的比较是不公平的。一种明显提速生成建议框的方法是在GPU上实现它，这是一种工程上很有效的解决方案，但这个方法忽略了其后的检测网络，因而也错失了共享计算的重要机会。
Faster R-CNN中使用深度网络计算建议框，建议框计算几乎不会给检测网络的计算带来消耗。
RPN网络主要分为ZF部分，和RPN部分

ZF Net part

  1. 首先，输入图片大小是 224×224×3
  2. 然后第一层的卷积核维度是 7×7×3(×96)，stride=2，padding=3
  3. 所以conv1得到的结果是110×110×96 (110=(224-7+pad)/stride+1)
  4. 3×3 max pool stride=2，得到pool1，池化后的维度是55×55×96 55=(110-3+pad)/stride+1；
  5. 然后接着就是再一次卷积，conv kernal size 5×5×96(×256), stride = 2，得到conv2：26×26×256；
  6. 最后获取conv5的输出，也就是13×13×256送给RPN网络的；

RPN part

  1. conv feature map的维度是13×13×256的；
  2. Faster R-CNN文章中指出，sliding window的大小是3×3的，如何得到这个256-d的向量呢? 只需要一个3×3×256(×256)这样的一个4维的卷积核，就可以将每一个3×3的sliding window 卷积成一个256维的向量；图里只是针对一个sliding window的；在实际实现中，我们有很多个sliding window，所以得到的并不是一维的256-d向量，实际上还是一个3维的矩阵数据结构。
  3. 然后就是k=9，所以cls layer就是18个输出节点了，那么在256-d和cls layer之间使用一个1×1×256×18的卷积核，就可以得到cls layer，当然这个1×1×256×18的卷积核就是大家平常理解的全连接；所以全连接只是卷积操作的一种特殊情况（当卷积核的大小与图片大小相同的时候，其实所谓的卷积就是全连接了）
  4. reg layer也是一样了，reg layer的输出是36个，所以对应的卷积核是1×1×256×36，这样就可以得到reg layer的输出了；
  5. 然后cls layer 和reg layer后面都会接到自己的损失函数上，给出损失函数的值，同时会根据求导的结果，给出反向传播的数据

MCDC 车道线检测

Posted on 2018-07-15

MCDC 车道线检测比赛

引入

今天对6月参加的Momenta公司举办的车道线检测比赛做个总结

  1. 数据集：训练集是分批次发放，2W->10W->30W，数据集的组织形式为images(.jpg) + points(.json)
  2. 基本的思路：使用语义分割模型，先预测出车道线像素的位置，然后使用聚类算法，将像素对应的坐标值进行聚类，得到具体的车道线。

基本流程和遇到的问题：

  1. 数据集存储在服务器上，需要远程访问，并且GPU也需要远程访问。
  2. 搭建远程服务，jupyter + ssh
  3. 具体配置：tensorflow-gpu:1.8.0 keras:2.1.6 GPU:8*TITAN
  4. 使用jupyter搭建模型爆显存问题：jupyter中运行模型结束后不会释放显存，需要在terminal中kill的相关的进程(jupyter同样不会释放相关变量占用的内存)

模型部分:

我们最终使用的Unet作为分割网络，对全局图像进行分割（后面提到这样做的局限性），预测出每个像素的mask，然后对mask->points进行聚类，聚类的结果为预测出来具体的线。

存在的问题：

  1. 由于在训练集中存在很少一部分Y型线，但test中存在的大量的Y型线，导致出现的过拟合
  2. 预测得到的mask有噪声点，或者团状的mask，单独将他们作为一条线是不合适的。
  3. 预测得到的mask存在粘连问题，即车道线在远地点会有汇合的趋势，这样就造成了，在聚类过程总可能会出现将不同线聚成一条的问题（DBScan cluster）

解决方案：

  1. 针对不同形状的线，我们使用了（0 直线,1 曲线, 2 Y型线, 3 无意义），建造一个额外的数据集，构造Y型线的分类器。
  2. 对于直线采取一次拟合的策略，并将其延伸到合适的位置；对曲线和Y型线，没有采取拟合的策略；略过无意义团状线
  3. 对于粘连问题我们采用了cut_top函数，对预测出来的mask图像顶端进行合理的裁剪

结果展示：

总结：

我们直接使用的Unet分割方案还是有一定的局限性，另外一个组使用的是先检测后分分割的思想，先使用faster rcnn进行车道线的定位，然后再使用分割网络对检测到的区域进行分割，取到了不错的效果。还有一组针对点聚类成线的过程使用了instance segementation的策略，有待学习。

关于FaceNet和Adaboost的一些想法

之前读了FaceNet关于人脸识别的这篇文章，以及学过的Adaboost算法，感觉这两者在思路上有相同之处。
Adaboost是通过改变样本的权值，加大难训练样本在的权重，来着重训练易分类错误的样本，从而达到较好的效果。
无独有偶，FaceNet的triple tuple的生成也是利用这样的思想，尽量寻找难分类的image组成的三元组，对模型进行训练，这样得到的参数会有更好的泛化能力。
idea都是相通的。

矩阵分解

  1. 奇异值分解（Singular Value Decomposition）是线性代数中一种重要的矩阵分解，是矩阵分析中正规矩酉对角化的推广。在信号处理、统计学等领域有重要应用。奇异值分解则是谱分析理论在任意矩阵上的推广。
  2. 应用
    a. 矩阵近似值：
    奇异值分解在统计中的主要应用为主成分分析（PCA），一种数据分析方法，用来找出大量数据中所隐含的“模式”，它可以用在模式识别，数据压缩等方面。PCA算法的作用是把数据集映射到低维空间中去。 数据集的特征值（在SVD中用奇异值表征）按照重要性排列，降维的过程就是舍弃不重要的特征向量的过程，而剩下的特征向量组成的空间即为降维后的空间。
    b. 求伪逆
    求伪逆通常可以用来求解线性最小平方、最小二乘法问题。

FunkSVD小解：先生成两个随机分解矩阵，然后讲这两个分解矩阵进行相乘，和原矩阵计算loss，这里用的是L2距离。然后进行矩阵更新，得到新的矩阵，矩阵更新的公式改天补上。整个算法中只有两个超参数，alpha，beta。
这里做了一些小改进，有时候在梯度下降达不到指定标准，但是有没有到迭代的指定次数的时候，经过多次迭代，会出现loss增加的情况，我们加了一个终止条件，在loss增加时，停止迭代。

  1. 结果
    经过FunkSVD函数的分解，我们得到两个矩阵，对这两个矩阵的相乘，得到对整个推荐矩阵。这个矩阵就系统过滤的结果，我们把一个稀疏矩阵进行分解，然后将结果相乘，得到一个“填满”的矩阵，然后根据这个填充好的矩阵里，用户对商品的rating，给用户的推荐相应的商品。
